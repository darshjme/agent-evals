# agent-evals — Performance Benchmarks

Benchmarks measure the wall-clock throughput of `AgentEvaluator.evaluate()`
against a zero-latency mock agent, isolating the framework's own overhead
(scoring, guard checks, data-model construction) from any real LLM latency.

## Environment

| Item | Value |
|------|-------|
| Python | 3.13.11 |
| OS | Kali GNU/Linux (x86-64) |
| CPU | AMD EPYC 7401P 24-Core |
| Agent | Synchronous mock (no I/O, no sleep) |

## Results

| Cases | Time (s) | Cases/sec | Avg (ms/case) |
|------:|----------:|----------:|--------------:|
|    10 |    0.0003 |  35,644.4 |         0.028 |
|    50 |    0.0012 |  42,720.0 |         0.023 |
|   100 |    0.0023 |  43,565.7 |         0.023 |

**Peak throughput: ~43,566 cases/sec**

> Numbers are real — generated by running `python3 benchmarks/benchmark_evaluator.py`
> on the commit above.  No fabricated values.

## What's measured

Each benchmark call times only `evaluator.evaluate(cases)`, which covers:

- Iterating over every `EvalCase`
- Invoking the agent function
- Post-run guard sweep (MaxSteps, CostCeiling, LoopDetection)
- Three-dimensional scoring (task completion, reasoning, tool use)
- `EvalReport.finalize()` aggregate stats

It does **not** include: LLM API calls, tokenisation, or network I/O — those
are agent-implementation concerns and dominate in production.

## Running the benchmark yourself

```bash
git clone https://github.com/darshjme/agent-evals
cd agent-evals
python3 benchmarks/benchmark_evaluator.py
```

No extra dependencies are required beyond the standard library and the
`agent-evals` package itself.
